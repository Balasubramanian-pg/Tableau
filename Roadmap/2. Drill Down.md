# 1 Introduction to Drill Downs in Tableau

Drill downs in Tableau allow users to move from a high level summary to successively deeper levels of detail. Tableau provides both automatic mechanisms, such as built in hierarchies, and fully customisable mechanisms, such as parameter or set actions. These features form the backbone of modern interactive dashboards. Verified behaviors of hierarchies, drill buttons, and action driven interactivity are documented in Tableau’s official guides.

---

# 2 Method A Creating Drill Downs with Built in Hierarchies

Tableau enables simple hierarchical drill downs using the plus and minus buttons next to fields on the Rows or Columns shelves.

## 2.1 When to Use Built in Hierarchies

Use this method when your data naturally forms a strict parent to child relationship such as Region to State to City or Year to Quarter to Month. Hierarchies provide native expand and collapse controls and preserve the logical structure of the data.

## 2.2 Step by Step Tutorial

1. Connect to your data source.
    
2. In the Data pane, drag one field onto another to create a hierarchy. Tableau prompts you to name the hierarchy.
    
3. Add the top level field of the hierarchy to Rows or Columns.
    
4. Tableau automatically displays a plus icon next to the field header. Select the plus icon to expand to the next level. Select the minus icon to collapse. These icons are official indicators of drill availability.
    
5. Add the worksheet to a dashboard if you want drill downs within a larger dashboard structure. The plus and minus behavior remains functional.
    

## 2.3 Practical Tips

• Built in hierarchies expand all members at once. This is by design.  
• Names in the hierarchy pane can be rearranged by dragging fields.  
• Performance is usually strong since Tableau optimises hierarchical aggregation.

# 3 Method B Creating Drill Downs with Dashboard Filter Actions

Instead of expanding the same view, this method filters another sheet when the user selects a value in the source sheet.

## 3.1 When to Use Filter Actions

Use filter actions when each drill level is represented by a separate worksheet. For example, clicking a Region filters another worksheet showing State level data. These actions are common in executive dashboards. Verified interface paths for adding filter actions are included in official Tableau documentation.

## 3.2 Step by Step Tutorial

1. Create a high level worksheet such as Sales by Region.
    
2. Create a detail worksheet such as Sales by State.
    
3. Open a dashboard and place both worksheets on the canvas.
    
4. Select Dashboard, then Actions, then Add Action, then Filter. Select the source sheet. Point the action to one or more target sheets.
    
5. Choose when the action runs. The run on Select mode is the most common.
    
6. Test by selecting a region. The target worksheet should update.
    

## 3.3 Practical Tips

• Multiple actions can layer interactivity but may confuse users if overused.  
• To clear the filter, deselect marks or create a clear button.

# 4 Method C Creating Drill Downs with Parameter Actions

Parameter actions allow a single click to change the value of a parameter. The parameter value then controls which rows appear.

## 4.1 When to Use Parameter Actions

Use this method when you want a smooth drill down without creating several worksheets. Parameter actions are excellent when you want a single view that changes based on user selection. Tableau provides official guidance on how parameter actions propagate values.

## 4.2 Step by Step Tutorial

1. Create a parameter named Selected Category and populate it with a list including a default such as All.
    
2. Create a calculated field named Category Filter
    
    ```
    [Selected Category] = "All" OR [Category] = [Selected Category]
    ```
    
3. Add this calculated field to Filters and keep True.
    
4. Add your worksheet to a dashboard.
    
5. Create a parameter action by selecting Dashboard then Actions then Add Action then Change Parameter. Map the parameter to the Category field.
    
6. Clicking a category sets the parameter, and the worksheet redraws to show only the selected category. Tableau’s documentation confirms that parameter actions execute immediately upon selection.
    

## 4.3 How Parameter Actions Improve UX

• Custom labels or icons can appear conditionally depending on the selected parameter value.  
• Parameter actions can imitate expand and collapse without the need for multiple worksheets.  
• They also support back navigation by resetting the parameter to All.

# 5 Method D Creating Multi level Drill Downs with Set Actions

Set actions allow the content of a set to change dynamically based on user interaction. This method offers the highest level of customisation.

## 5.1 When Set Actions Are Best

Use this method when your drill down requires multiple expansion states at different levels and where preserving the user's selections matters. Verified references confirm that set actions can dynamically include or exclude members based on marks selected.

## 5.2 Step by Step Tutorial

1. Create a set called Category Set based on the Category field.
    
2. Create calculated fields that reference whether a value is in the set. These calculations determine visibility and indentation.
    
3. Create a worksheet using these fields. Often a string field is used to include symbols such as arrows or indentation for hierarchy.
    
4. Add the worksheet to a dashboard.
    
5. Go to Dashboard then Actions then Add Action then Change Set Values. Choose the Category Set as the target.
    
6. Test: selecting a category adds or removes the category from the set, causing expansion or collapse.
    
7. To build multiple levels such as Category to Sub Category to Product, repeat with additional sets and nested calculations inspired by community practices that demonstrate multi level set driven drill downs.
    

## 5.3 Advanced UX Patterns

• Some designers create a text based tree table with Unicode arrows that flip when expanded.  
• Nested sets can maintain a memory of open branches, making complex trees user friendly.  
• Conditional formatting can highlight expanded paths.

# 6 Suggested Workbook Structure for Drill Down Dashboards

• Worksheet 1 Summary view showing the highest level of aggregation  
• Worksheet 2 Hierarchical view using either built in hierarchy or set action logic  
• Worksheet 3 Detailed transactional view  
• Dashboard 1 Combines all sheets with appropriate filter, parameter, or set actions

This modular structure allows you to test and swap drill mechanisms without redesigning everything.

---

# 7 Troubleshooting Guide

• If the plus icons do not appear, ensure your fields are in a defined hierarchy.  
• If filter actions fail, confirm the selected run mode is correct and that the target sheets accept filters from the source fields.  
• If set actions behave inconsistently, ensure the set is not filtered out by other dashboard elements.  
• If parameter actions do nothing, confirm the parameter is actually referenced inside calculations used in the view.
